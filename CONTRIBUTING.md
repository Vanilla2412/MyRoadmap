# Contributing to My Roadmap

Thank you for your interest in contributing to My Roadmap! As this is a portfolio project focused on high-level engineering skills, we follow professional standards for development.

## Development Workflow

1.  **Issue First**: Before starting any work, please ensure there is an open issue describing the task.
2.  **Branching Strategy**:
    *   Main branch: `main` (Protected, requires PR for all changes)
    *   Feature/Fix branches: `feat/xxx`, `fix/xxx`, `chore/xxx`
3.  **Pull Requests**:
    *   Always target `main`.
    *   Provide a clear description of changes.
    *   Ensure CI checks pass.

## Commit Message Policy

We follow [Conventional Commits](https://www.conventionalcommits.org/). This allows us to automate release notes.

Format: `<type>(<scope>): <description>`

Common types:
*   `feat`: A new feature
*   `fix`: A bug fix
*   `docs`: Documentation changes
*   `style`: Formatting, missing semi colons, etc; no code change
*   `refactor`: Refactoring production code
*   `test`: Adding missing tests, refactoring tests; no production code change
*   `chore`: Updating build tasks, package manager configs, etc; no production code change
